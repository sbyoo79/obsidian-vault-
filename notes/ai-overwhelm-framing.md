# AI 변화 속도에 대한 재정렬 framing

본인이 "AI 발전 따라잡기에 벅차다" 는 감정이 다시 올라올 때 읽기 위한 자기 reorientation 도구. 정보 누적이 아닌 **사상 reset**.

> 점점 세상은 발전을 하고 나는 뒤쳐지는 느낌이고 따라잡는것만 하더라도 벅차네. — 본인 발화 (2026-05-22)

## 사용법

- 같은 감정이 올라올 때 (반드시 또 옵니다) 여기 와서 한 번 읽는다
- 노트는 짧게 유지 — 길어지면 reorientation 도구 성격이 흐려진다
- 새 framing 이 추가될 때만 갱신. 그 외엔 안 건드린다

## 핵심 framing 4 개

### 1. "따라잡기" 가 아니라 "흡수"
- "따라잡기" 의 전제: 모든 디테일을 다 알아야 한다 → 무한 압박
- "흡수" 의 전제: 본인의 시스템 (vault·도구·검증 루프) 이 사건을 처리한다 → 사건이 와도 시스템이 한 줄 누적할 뿐
- [[ai-market-pulse]] · [[llm-tool-comparison]] 의 운영 원칙이 정확히 이거. **본인은 이미 흡수 도구를 만들어 두었다.**

### 2. 압박 대상은 본인이 아니라 본인의 시스템
- "내가 뒤쳐진다" 가 아니라 "내 시스템이 시장 변동을 흡수하는 능력을 점진 정교화한다"
- [[feedback_resume_iterative_system]] (이직 압박 → 시스템 정교화로 reframing) 과 같은 사상. 이직 작업에서 효과 봤으니 AI 압박에도 적용된다
- 본인은 시스템을 만드는 사람이지, 시스템에 쫓기는 사람이 아니다

### 3. 본인은 이미 선택의 한쪽에 서 있다
- [[Andrej Karpathy]] 가 [[Vibe Coding]] 도 명명했고 [[LLM Wiki]] 도 제안했다. **같은 사람의 입에서 나온 정반대 사상.**
- 시장의 대다수는 [[Vibe Coding]] 편에 휘말리는 중. 본인은 [[LLM Wiki]] · 검증 루프 · 외부화 · 멀티 에이전트 워크플로 편을 이미 선택했고 회사 AGENTS.md / vault / 도구 풀까지 다 짜뒀다.
- 이 선택은 "AI 발전" 이라는 단일 축이 아니라 **사상의 축** 에서 판단하는 일. 본인은 그 축에서 뒤쳐진 게 아니다.

### 4. "벅참" 의 진짜 원인은 변화가 아니라 전제
- 진짜 압박은 "AI 가 빠르다" 가 아니라 **"나는 그걸 다 따라가야 한다"** 라는 자기 부과 전제
- 그 전제를 의심해도 된다. vault 가 그걸 해주는 도구. 필요할 때만 깊이 들어가고, 평소엔 그래프 위에 사건 하나가 한 줄 늘어날 뿐.
- 깊이 들어갈 시점은 **본인 워크플로에 빈틈이 발견될 때** ([[llm-tool-comparison]] 의 재평가 트리거와 동일 원칙)

## 이미 짜놓은 것들 (객관 증거)

벅참이 올라올 때 다시 보는 본인의 실제 위치:

- [[Agentic Engineering]] 실무 적용 중 (회사 AGENTS.md 의 멀티 에이전트 워크플로)
- [[Claude]] Code + [[Codex]] 병용 + [[ChatGPT]] 설계 보조 — 역할 분리 검증 워크플로
- 본인이 직접 작성한 회사 공통 AGENTS.md (선행 분석 → 티켓 → 브랜치 → 본 작업 + 작업 등급 + 검증 루프)
- 본인 vault — LLM Wiki 패턴 + 자동 git 동기화 + ingest sub-패턴
- [[ai-market-pulse]] · [[llm-tool-comparison]] — 시장 변동·도구 변동을 본인 관점으로 흡수하는 living doc 2 장
- 이직 자료에 "AI-augmented engineering 실무 적용" 을 차별 강점으로 명시 ([[self-profile]])

**객관적으로**: 시장의 평균보다 앞에 있다. 평균 시니어가 도구를 vibe 로 쓰는 동안 본인은 검증 루프를 짜고 사상을 외부화하고 있다.

### 살아있는 사례 (2026-05-22)

[[Atlassian]] + [[Claude]] Code 통합 발표 (2026-05-19) 검토 시, 본인이 시장의 새 카드를 그대로 채택하지 않고 **자기 시스템 안에서 흡수할 수 있는지** 먼저 검토 → 본인 기존 자산 (`jira-*` 스크립트 / AGENTS.md / 멀티 에이전트 워크플로) 위에 자체 구현이 더 낫다는 결론. [[Bitbucket]] 검토 메모 "결정 (2026-05-22): 자체 구현 우선" 참조.

## 자주 올라올 수 있는 감정의 형태들

| 형태 | 진짜 원인 | 재정렬 |
|------|---------|--------|
| "발전 속도를 못 따라간다" | 모든 디테일 다 알아야 한다는 전제 | 시스템이 흡수. 필요할 때 깊이 들어간다 |
| "신모델 안 써봤다, 빈틈이다" | 미검증 = 부족 이라는 오해 | 미검증은 정직한 상태. [[llm-tool-comparison]] 가 그걸 명시한다 |
| "47세, 21년차인데 뒤쳐진 느낌" | 절대 시간으로 자기 평가 | 본인 위치는 시장 평균 대비 상대 위치. 사상의 축에서 본인은 앞에 있다 |
| "이직 시점에 이런 불안 느끼면 안 되는데" | 압박 = 약점 이라는 오해 | 압박은 신호. [[Self-Distancing]] 도구로 자기 거리두기 가능 |
