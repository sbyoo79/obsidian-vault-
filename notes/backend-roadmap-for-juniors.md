# 주니어 백엔드 추천 자료 — 골든래빗 백엔드 로드맵 + 시니어 보강

[[yozm-backend-roadmap]] ingest 하며 정리한 **21 년차 시니어 관점** 노트. 후배·주니어에게 추천하는 자료 + 본인 경험으로 보강하는 영역으로 구성.

## 왜 이 자료가 좋은가

1. **체계적 우선순위** — OS → 네트워크 → 웹/WAS → 프레임워크 → 아키텍처 → 언어 → DB → 클라우드 → DevOps. 학습 순서가 명확
2. **진입장벽 현실 인식** — "자바·JSP·SQL 만으로 백엔드 가능" 시절과 다르다는 사실 명시. 후배들이 빨리 인지해야 할 시장 환경
3. **출판사 자료** — [[골든래빗]] 의 정석 자료. 신뢰성·일관성
4. **무료 + 짧음** — 한 번에 훑기 좋은 분량

## 시니어 보강 영역 (기사가 약하거나 빠진 부분)

본 기사는 2023 년 기준이라 2025~ 시점에서 다음 영역이 부족.

### 1. AI-augmented Engineering (2025~)

본인의 차별 포지셔닝 ([[self-profile]] 참조). 2023 년엔 부각 안 됐지만 2025~ 시점에선 **반드시 학습 영역**.

- 멀티 에이전트 역할 분리 ([[Agentic Engineering]] 흐름)
- 단순 [[Vibe Coding]] 회피 — AI 가 쓴 코드를 검증 없이 받아들이지 않는다
- 컨텍스트 엔지니어링 ([[Context Engineering]]) — KB / docs 우선 참조
- 검증 루프 (cross-LLM 리뷰, 사람 검증) 가 동반된 운영 방식

→ 본인 사례: GPT 설계 / Codex 구현 / Claude 리뷰 멀티 에이전트 워크플로로 광고 플랫폼·DW 1 주 내 구축

### 2. DB 설계 + 튜닝 깊이

기사가 DB **종류 비교** 에 집중하지만, 시니어 강점 3 축 중 하나는 **DB 설계 + 튜닝** (자기 평가 레벨 4).

- 단순 ORM 사용에서 멈추지 말 것 — 실행 계획 / 인덱스 / 쿼리 튜닝 직접 경험 필요
- 멀티테넌트 / 정합성 / 보정 도메인의 DB 설계는 시간 들여 쌓이는 자산
- 본인 강점 3 축: Java 4 + Spring/Boot 4 + **DB 설계+튜닝 4** — 다른 21 년차 시니어와 차별되는 부분

### 3. 위험·복잡 마이그레이션 경험

기사는 "기술을 안다" 에 집중하지만 시니어 가치는 **"위험을 안전하게 끝낸다"** 에서 나온다.

- 본인 사례: Java 1.8 → 17 (Spring Boot 1.4.2 → 2.7.18), 200 인스턴스 → 단일 멀티테넌트, 3.2 PB 윈도우 → 리눅스 컨버전
- 후배 단계에선 직접 경험 못 하지만, 시니어 인터뷰 / 멘토링 / 커리어 방향 잡는 데 도움

### 4. 0 → 1 + 종단 책임 경험

기사가 다루지 않는 영역. 본인의 차별 포지셔닝 #1.

- MAXVISOR (2016~2019) 에서 0→1 설계자가 7~9 년 후 후속 회사 리드플레닛에서 그 시스템을 직접 개선
- 한 시스템을 길게 본다는 시각을 일찍 가지면 좋음

### 5. 문서화 + 인계 가능성

기사가 다루는 개발 생명주기 (분석 → 개발 → 테스트 → 배포 → 유지보수) 에 빠진 항목 = **인계 가능 문서화**.

- 작업 완료 = 코드 + 검증 + 문서 + 인계 가능 상태 (4-tuple)
- 시니어가 AI 와 동반해서 자산화하는 영역

## 추천 흐름 (주니어 → 시니어 단계별)

| 시기 | 본 기사 우선 학습 | 시니어 보강 |
|------|---------------|----------|
| **0~1 년차 (입문)** | OS / 네트워크 / 웹·WAS / 프레임워크 / 언어 1 개 | AI 도구 기본 사용법 ([[Claude]] / [[Cursor]] / [[ChatGPT]]) — 단 [[Vibe Coding]] 회피 인지부터 |
| **1~3 년차 (실무)** | 아키텍처 / DB / DevOps | DB 설계·튜닝 깊이 / 검증 루프 도입 / 작은 마이그레이션 경험 |
| **3~5 년차 (시니어 진입)** | 클라우드 / MSA | 멀티 에이전트 워크플로 / 마이그레이션·리뷰 책임 |
| **5~10 년차 (시니어 / 리더)** | — | 0→1·종단 책임 / 팀 리딩 / 인계 가능 문서화 / 외부 발표 |
