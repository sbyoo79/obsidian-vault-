## 약점 / 학습 필요 / 리스크

### Kafka — 학습 필요 영역

- 본인 미사용. master 이력서에 정직 표기.
- 빅데이터 인프라 (실시간 분석) 의 일부 요건. Aurora MySQL + Redis 는 OK 지만 Kafka 가 빈틈
- 학습 곡선: Java + MQ (RabbitMQ / SQS) 운영 경험 있어서 진입 빠를 것
- 대응: "MQ 운영 경험 + Outbox 패턴 운영 → Kafka 학습 진입 빠름. 회사 환경 적응 시 학습 의지 충분"

### Go — 선택지로 회피

- "Go, Java/Kotlin 중 최소 1 개" → Java 로 요건 충족
- 다만 회사 Go 도입률이 높으면 실무에서 학습 부담
- 면접에서 확인: "Go 신규 / Java 레거시 / Kotlin 비중" 현황

### Kubernetes — 우대지만 필수 X

- 누아 (필수) 와 달리 본 공고는 우대. 부담 ↓
- 동일 자기 평가: "개념 이해 O, 운영 X, 회사 필요 시 학습 OK"

### 금융 / 마이데이터 도메인 — 직접 경험 X

- **본인 가장 큰 약점** in 본 케이스. 본격 매칭이지만 도메인 격차
- 인접 경험:
  - 결제 시스템 연동 (KICC / 스마트로 / 메타시티 / 키움 가상계좌, MAXVISOR)
  - 정산 / 매출 / 물류 데이터 처리 (리드플레닛 DW)
  - 글로벌 회원 시스템 (멜론 티켓 글로벌)
- 대응: 도메인 학습 의지 명시 + 인접 경험에서 데이터 안정성·정확성·결제 흐름 이해 부각
