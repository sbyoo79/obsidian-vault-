## 약점 / 학습 필요 / 리스크

### Docker / K8s — 가장 큰 변수

- 본인 [[infra-tradeoff-framing]] 에서 **K8s 는 의도적 회피 영역** 으로 박혀 있음. 운영 부담 관찰 기준 회피 결정
- 공고는 **필수 요건** 으로 명시 — "사용 경험" 정도가 어느 수준인지가 매칭 결정의 핵심 변수
- 본인 사용 경험 정도 점검 필요:
  - Docker: 로컬 개발 환경 / 단순 컨테이너 빌드 정도라면 OK
  - K8s: **본격 운영 경험 부족** = 솔직한 자기 평가. 면접에서 거짓말 X
- 매칭 도구의 [[infra-tradeoff-framing]] 적용:
  - "여유 = 시간 + 인지 부하 모두 충족" 조건 — 면접 통과 후 입사 전 학습 기간 확보 가능한지
  - **회피 ≠ 영구 거부 원칙** — 새 회사 환경이 학습 의지·여유 조건 충족 트리거가 될 수 있음
- 솔직한 대응: 자기 평가 그대로 어필 — "Docker 기본 사용 경험은 있으나 K8s 본격 운영은 제한적. 21 년차 시니어로 학습 속도 빠르므로 입사 시 빠르게 보강 가능"

### 연봉 미공시

- 본인 하한 1억 1,000 만원. 흑자 + 130 억 투자 회사라 가능성 있음
- 협의 단계에서 명시. 미공시 자체가 부정 신호 X
- 합격·입사 시 50 만 보상금 (작은 plus)

### 통근

- 부평 → 서울 중구 = 1 호선 직통 약 1h~1h10. 본인 통근 한계 (1h30) 내. 다만 출근형 정규직 추정 → 매일 통근 부담
- 금요일 점심 1h30 + 자유 연차 문화는 plus

### 영어 / 글로벌

- 회사가 일본·중화권·동남아 진출 준비 중 = **점진 글로벌**. 본인 영어 약점 보완 의지와 결이 맞음 (입사 후 점진 노출)
- Fieldguide 처럼 즉시 영어 환경 부담 X — 한국어로 시작, 점진 글로벌
