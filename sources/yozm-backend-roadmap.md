# 누구도 알려주지 않는 백엔드 로드맵

[[골든래빗]] 의 [[YozmIT]] 기고. 백엔드 개발자가 알아야 할 광범위한 영역을 운영체제·네트워크·웹/WAS·프레임워크·아키텍처·언어·DB·클라우드로 나눠 체계 정리한 입문 가이드. **자바·JSP·SQL 만으로 충분하던 시대는 끝났다** 는 인식에서 출발해, "무엇부터 배워야 하는가" 의 학습 우선순위를 제시.

## 다루는 학습 영역

- **서버 운영체제** — Linux 우선
- **네트워크** — HTTP / DNS
- **웹 서버 / WAS** — Nginx / Apache / Tomcat 역할 분담
- **프레임워크** — Spring / Django / Express / NestJS / Flask / FastAPI
- **개발 생명주기** — 과제 할당 → 분석 → 개발 → 테스트(리뷰) → QA 및 버그 수정 → 배포 → 유지보수
- **아키텍처** — 계층형 / 이벤트 기반 / 마이크로서비스 3 가지 비교
- **언어** — Java / Python / JavaScript / TypeScript / C# / C++ (백엔드 주력 비교)
- **DB** — RDB (MySQL / PostgreSQL / Oracle) + NoSQL (MongoDB / Redis / DynamoDB / Cassandra)
- **클라우드** — AWS / Azure / GCP 3 사 비교
- **DevOps** — Git/GitHub / Docker / Kubernetes / CI·CD

## 핵심 메시지

> "자바, JSP, SQL만 알아도 백엔드 개발자가 될 수 있던 시절이 있었습니다. 오늘날에는 백엔드 개발자가 되려면 알아야 하는 지식이 너무나 많습니다."

> 백엔드 개발자 = "서비스에 데이터라는 피가 흐르게 해주고 정보라는 숨을 불어넣어주는 사람"

## 통계 / 사실 인용

- 2022 1Q 기준 **AWS / Azure / GCP 3 사 클라우드 시장 65%** 점유
- TIOBE 인덱스 기준 백엔드 주력 언어: Java / Python / C++ / C# / JavaScript
